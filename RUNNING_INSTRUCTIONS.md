## 2Ô∏è‚É£ Install dependencies

Open a terminal in the project root (`campus_sentinel`) and run:

```bash

```

This will read `pnpm-lock.yaml` and install **all** required packages, including `next`, `react`, `radix-ui`, etc.

> **Why pnpm?**
> pnpm creates a content‚Äëaddressable store and links packages, which makes installation faster and uses less disk space. The lockfile is already in pnpm format, so `pnpm install` guarantees the exact same versions the project was built with.

## 5Ô∏è‚É£ Common pitfalls & fixes

| Problem                                          | Fix                                                                                                                                                       |
| ------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `npm run dev` says **"next: command not found"** | Make sure you ran `pnpm install` (or `npm install`) first. The `next` binary lives in `node_modules/.bin`.                                                |
| **Port 3000 already in use**                     | Either stop the other process or run the dev server on a different port: `pnpm dev -- -p 3001`                                                            |
| **TypeScript build errors**                      | The project deliberately sets `ignoreBuildErrors: true` in `next.config.mjs`. You can still run the dev server; the errors are only shown in the console. |
| **Missing `node_modules` folder**                | Delete `pnpm-lock.yaml` and run `pnpm install` again, or run `pnpm install --force` to rebuild the store.                                                 |
